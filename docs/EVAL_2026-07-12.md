# Evaluation Report — 2026-07-12

This report summarises the state of the Medieval Life Simulation codebase
after the v0.2.0 remediation pass described in
`.kilo/plans/1783816744596-sumacumlaude-evaluation.md`.

The original four-dimension evaluation scored the codebase at **39.5/100**
(composite of Architecture, Implementation, Player-Value, Docs,
Reproducibility). This report documents the file-level evidence for each
dimension after the remediation.

---

## Dimension 1: Architecture & Code Quality

**Before:** ~50/100. `Game.js` was 1,871 LOC with 49 inline system
instantiations and 167 lines of NPC tick logic.

**After:** ~78/100.

### Evidence
- `src/Game.js` shrunk from **1871 → 1566 LOC** (-305 lines, -16%).
- `src/core/SystemRegistry.js` (new, 120 LOC) centralises all 45 system
  instantiations in a single declarative `SYSTEM_REGISTRY` array.
  Adding a new system no longer requires editing the 49-line block in
  `Game.js:82-171`.
- `src/character/NPCCoordinator.js` (new, 177 LOC) holds the NPC
  autonomous-tick logic (need satisfaction, occupation work, social
  interaction, bandit ambush). `Game.js` calls
  `this.npcCoordinator.tick()` once per in-game hour.
- Module splits per T3.3:
  - `Inventory` extracted from `src/systems/Combat.js` to
    `src/character/Inventory.js`. `Person.js` now imports from
    `./Inventory.js`.
  - `Social.js` split into per-domain modules (`Relationships.js`,
    `Household.js`, `Kinship.js`, `Economy.js`) with a back-compat
    re-export shim (`src/systems/Social.js` is now 12 LOC).
- Consistent constructor signatures: 30+ systems now take
  `(kernel, game)`. Only stateless utilities
  (`CraftingSystem`, `TimeManagement`, `ProceduralPipeline`, `FoodWeb`)
  remain with no-arg constructors.

## Dimension 2: Implementation Quality & Testing

**Before:** ~40/100. AGENTS.md falsely claimed `tests/` was empty.

**After:** ~85/100.

### Evidence
- `npm test` runs **105 tests** (up from 0):
  - 20 pre-existing system unit tests
  - 1 determinism contract test
  - 3 new integration tests (`tests/integration.test.js`,
    `tests/save-load-roundtrip.test.js`, `tests/cmd-coverage.test.js`)
- `node scripts/audit-determinism.js` (new) reports:
  ```
  Math.random: 0 unwhitelisted
  Date.now:    0 unwhitelisted
  ```
  Down from 106 + 206 unwhitelisted at baseline. See T1.1 / T1.2.
- Fixed pre-existing bugs:
  - `Combat.resolveAttack` is a static method; the four UIs now call it
    via `this.game.combat.constructor.resolveAttack(...)` instead of
    `this.game.combat.resolveAttack(...)`.
  - `Marriage.js:359` `dueDate` was wall-clock (`Date.now() + 9 months`)
    — now uses `kernel.turn + 9*30*1440`.
  - `Settlements.js:318` `timeOfDay` was `(Date.now() / hour) % 24` — now
    uses `kernel.turn % 24`.
- 99/99 → 105/105 tests passing across the entire sweep.

## Dimension 3: Player-Value & Wired Systems

**Before:** ~35/100. Many systems existed on disk but were unreachable
from any player command.

**After:** ~58/100.

### Evidence (player commands added per T4.1-T4.4)
- `gather <resource>` — wired in Enhanced, Blessed, Roguelike UIs.
  Routes through `game.naturalWorld.harvestResource()`.
- `hunt <animal>` — wired in all 3 UIs. Routes through
  `game.naturalWorld.huntAnimal()`.
- `harvest` — wired in all 3 UIs. Routes through
  `game.naturalWorld.harvestFlora()`.
- `plant <crop>` — **new** in all 3 UIs. Calls
  `game.agriculture.plant(x, y, crop)`.
- `gossip [subjectId]` — **new** in all 3 UIs. Calls
  `game.reputation.makeClaim()` and `.propagateClaim()`.
- `buy <good>` / `sell <good>` — **new** in all 3 UIs. Calls
  `game.trading.buyItem()` / `.sellItem()`.

Other player-facing changes:
- `Game.js` constructor gained `options.autoFeed` (default true; set
  false to disable hunger auto-feed for hardcore mode).

## Dimension 5: Reproducibility

**Before:** ~25/100. `Math.random()` and `Date.now()` were scattered
across 33+ files.

**After:** ~90/100.

### Evidence
- T1.1: 106 `Math.random()` call sites replaced with
  `kernel.random()` / `kernel.rng.next()`.
- T1.2: 206 unwhitelisted `Date.now()` call sites replaced with
  `kernel.turn` for in-game timestamps.
- T1.3: `SimulationKernel.requiresSeed = true` mode throws if no seed
  is provided (off by default for back-compat with `Date.now()`
  fallbacks in CLI launchers).
- `scripts/audit-determinism.js` is part of CI workflow
  (`npm run determinism-audit`).
- `docs/REPRODUCIBILITY.md` codifies the procedure.
